# Few-shot Hangul Font Generation using Conditional DDPM

COSE362 Machine Learning Final Project, Spring 2026

This repository contains the final project code for small-scale Hangul font generation. The goal is to generate target-style Korean glyph images from a content glyph and a small set of target-style reference glyphs using a Conditional DDPM, with a Direct U-Net baseline for comparison.

## Quick start for graders

This project was designed to run in **Google Colab with a GPU runtime**.

### 1. Open the notebook from GitHub

Open this notebook in Google Colab:

```text
notebooks/hangul_font_generation_conditional_ddpm_colab.ipynb
```

Colab URL pattern:

```text
https://colab.research.google.com/github/<Github-ID>/20261R0136COSE362/blob/main/notebooks/hangul_font_generation_conditional_ddpm_colab.ipynb
```

### 2. Set the runtime

In Colab:

```text
Runtime → Change runtime type → GPU
```

### 3. Run the first setup cell

When a notebook is opened directly from GitHub in Colab, Colab loads only the notebook file. It does **not** automatically clone the whole repository. Therefore, the first notebook cell contains this setup block:

```python
REPO_URL = "https://github.com/<Github-ID>/20261R0136COSE362.git"
PROJECT_DIR = "/content/20261R0136COSE362"
```

Replace `<Github-ID>` with the submitted GitHub ID, then run the first setup cell. The setup cell clones the full repository into:

```text
/content/20261R0136COSE362
```

This is required because the notebook imports:

```text
src/hangul_font_diffusion_mvp.py
```

### 4. Prepare font files in Google Drive

The original font files are not included in this repository due to file-size and license considerations. To reproduce the same type of experiment, place Korean font files under:

```text
/content/drive/MyDrive/ML fonts/fonts/
```

Expected Drive layout:

```text
/content/drive/MyDrive/ML fonts/
└── fonts/
    ├── Nanum handwriting or other Korean font files (.ttf/.otf/.ttc)
    └── ...
```

The exact font list used in our experiment is documented in [`docs/dataset.md`](docs/dataset.md).

### 5. Run the notebook cells in order

The notebook performs:

1. package installation and repository setup
2. Google Drive mount
3. source import
4. font file copy and glyph rendering
5. metadata loading and jamo coverage check
6. Direct U-Net baseline training
7. Conditional DDPM training
8. sample glyph generation
9. quantitative evaluation with L1, MSE, RMSE, SSIM, LPIPS, and FID

Generated files are written under:

```text
/content/hangul_font_mvp/
├── rendered/
├── fonts/
└── outputs/
```

## Main result summary

The final model is a Conditional DDPM conditioned on:

- a content glyph from the base font,
- 10 target-style reference glyphs,
- and the noisy glyph image at each diffusion timestep.

The project compares the DDPM against a Direct U-Net baseline. Prior work such as pix2pix, CKFont, DM-Font, Diff-Font, and DK-Font is summarized in the project documentation. See [`docs/final_project_summary.md`](docs/final_project_summary.md) for a concise project summary.
